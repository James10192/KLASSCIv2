Assertions — With Skill (3/3)
✅
Identifie le problème : background sur <tr> ignoré par DomPDF
"DomPDF ne rend pas background-color quand la propriété cible un élément <tr>"
✅
La correction cible les <td>
.section-header td, .subject-row:nth-child(even) td, .summary-row td
✅
Ne suggère pas flexbox
Aucune mention de flex. Correction purement CSS selector.

Fix: background-color sur <tr> — With Skill

Cause : DomPDF v2.0 ne rend pas background-color ciblant un <tr>. Le background doit toujours être sur les enfants <td>.

/* ❌ WRONG */
.section-header { background: {{ $pdfPrimary }}; }
.subject-row:nth-child(even) { background: #f8fafb; }

/* ✅ CORRECT */
.section-header td { background-color: {{ $pdfPrimary }}; }
.subject-row:nth-child(even) td { background-color: #f8fafb; }
.summary-row td { background-color: #e5e7eb; }

Pattern général :

/* ❌ ignoré */    .header-row { background-color: #0453cb; }
/* ✅ fonctionne */ .header-row td { background-color: #0453cb; }

Fix: background-color sur <tr> — Baseline

Cause : DomPDF ne supporte pas background-color sur <tr>. De plus, :nth-child() n'est pas supporté.

/* APRÈS */
.section-header td { background-color: {{ $pdfPrimary }}; }
.subject-row-even td { background-color: #f8fafb; }
.summary-row td { background-color: #e5e7eb; font-weight: 700; }

Et dans Blade, pour remplacer :nth-child :

{{-- AVANT --}}
<tr class="subject-row">

{{-- APRÈS — Blade $loop->even remplace :nth-child --}}
<tr class="subject-row{{ $loop->even ? ' subject-row-even' : '' }}">

Bonus : identifie aussi l'incompatibilité :nth-child et propose $loop->even — insight supplémentaire par rapport au with-skill.
