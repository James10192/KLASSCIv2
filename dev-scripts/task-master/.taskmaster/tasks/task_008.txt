# Details:
1. Créer les événements principaux:
   ```php
   // Exemple d'événement pour paiement reçu
   class PaiementRecu implements ShouldBroadcast {
       use Dispatchable, InteractsWithSockets, SerializesModels;
       
       public $paiement;
       
       public function __construct(Paiement $paiement) {
           $this->paiement = $paiement;
       }
       
       public function broadcastOn() {
           return new PrivateChannel('comptabilite');
       }
       
       public function broadcastWith() {
           return [
               'id' => $this->paiement->id,
               'montant' => $this->paiement->montant,
               'etudiant' => $this->paiement->etudiant->nom_complet,
               'timestamp' => now()->toIso8601String()
           ];
       }
   }
   ```

5. Implémenter les notifications Laravel:
   ```php
   class PaiementNotification extends Notification {
       use Queueable;
       
       protected $paiement;
       
       public function __construct(Paiement $paiement) {
           $this->paiement = $paiement;
       }
       
       public function via($notifiable) {
           return ['mail', 'database'];
       }
       
       public function toMail($notifiable) {
           return (new MailMessage)
               ->subject('Nouveau paiement reçu')
               ->line('Un paiement de ' . $this->paiement->montant . ' a été reçu.')
               ->action('Voir détails', url('/esbtp/comptabilite/paiements/' . $this->paiement->id))
               ->line('Merci d\'utiliser notre application!');
       }
       
       public function toDatabase($notifiable) {
           return [
               'paiement_id' => $this->paiement->id,
               'montant' => $this->paiement->montant,
               'etudiant' => $this->paiement->etudiant->nom_complet
           ];
       }
   }
   ```
