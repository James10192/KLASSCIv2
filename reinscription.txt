  ⎿  Plan : Système de Réinscription avec Gestion des Passages et Redoublements

     🎯 Objectif

     Créer un système de réinscription automatisé qui :
     - Analyse les résultats de l'année précédente
     - Détermine automatiquement les passages en classe supérieure ou redoublements
     - Gère la réinscription avec les bonnes classes/niveaux
     - Maintient l'historique académique

     📋 Composants à Développer

     1. Service de Réinscription (ReeinscriptionService)

     - Analyser les résultats d'un étudiant pour une année donnée
     - Calculer les moyennes générales et par matière
     - Appliquer les règles de passage (seuils de moyenne, matières obligatoires)
     - Déterminer le niveau suivant ou redoublement
     - Gérer la création automatique des nouvelles inscriptions

     2. Modèle de Règles Académiques (ESBTPRegleAcademique)

     - Stocker les seuils de passage par filière/niveau
     - Gérer les matières éliminatoires
     - Définir les conditions de redoublement
     - Paramétrer les règles spécifiques (compensations, rattrapages)

     3. Contrôleur de Réinscription (ESBTPReinscriptionController)

     - Interface d'administration pour lancer les réinscriptions
     - Visualisation des étudiants éligibles au passage/redoublement
     - Validation manuelle des cas limites
     - Génération des rapports de réinscription

     4. Interface de Gestion

     - Dashboard pour visualiser les statistiques de passage
     - Formulaires de configuration des règles académiques
     - Interface de validation des réinscriptions en lot
     - Rapports détaillés par filière/niveau

     5. Migration de Base de Données

     - Table esbtp_regles_academiques pour les règles de passage
     - Champs additionnels dans esbtp_inscriptions pour le suivi
     - Index et contraintes pour optimiser les requêtes

     6. Système de Notification

     - Notifications aux étudiants (passage/redoublement)
     - Alertes aux coordinateurs pour validation
     - Emails de confirmation de réinscription

     🔄 Workflow Proposé

     1. Fin d'année : Calcul automatique des résultats finaux
     2. Analyse : Évaluation des conditions de passage pour chaque étudiant
     3. Proposition : Génération automatique des propositions de réinscription
     4. Validation : Révision et validation par les coordinateurs
     5. Exécution : Création des nouvelles inscriptions
     6. Notification : Information des étudiants et familles

     📊 Règles de Gestion

     - Passage normal : Moyenne générale ≥ 10/20 et aucune note éliminatoire
     - Redoublement : Moyenne < 10/20 ou échec dans matières fondamentales
     - Cas spéciaux : Gestion des compensations et décisions du conseil de classe
     - Limites : Maximum 2 redoublements par niveau (configurable)

     🛠️ Technologies

     - Backend : Laravel avec services métier
     - Frontend : Blade + Bootstrap pour l'interface d'administration
     - Base de données : Extension du schéma existant
     - Rapports : Génération PDF avec DomPDF
     - Notifications : Système de mail Laravel
